Rev 2 design principles (locked in chat 2026-05-12):

1. Pin / unpin top band (default pinned). Pinned = sticky top band, JobList scrolls underneath. Unpinned = whole page scrolls as one. State persists per user.

2. Single filter bar. The JobList toolbar drives map markers, SmallestAreaCard metrics, and JobList rows — one source of truth. Hover a row → matching marker pulses. Filter input debounced at ~150ms.

3. Two save verbs. "Save as Job List" persists a filter set; "Save to My Postings" persists a single posting. A provenance toast confirms either in plain language ("Saved Chicago–Naperville locality, FEMA, GS-13–14 as a new Job List").

4. Metrics inside SmallestAreaCard. No standalone Metrics surface. Four top-line blocks (Postings / Workforce / Pay vs COL / Urgency), each click-to-expand inline.

5. Page purposes. Browse = "find jobs to apply to, right now" (decisional). Map-only = "see where federal hiring is happening" (exploratory). Localities = "compare places against each other" (comparative).

6. Area summary stays deterministic. Templated prose from local fields. No Claude API per request, ever. If we want richer narration, it's a nightly batch over all localities — never on-demand.
